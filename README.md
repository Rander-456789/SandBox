# RecSys Music Sandbox

RecSys Music Sandbox — это MVP музыкального стримингового сервиса-песочницы для тестирования рекомендательных алгоритмов в реальном времени.

Основная идея проекта — не построить полноценный музыкальный продукт, а создать инфраструктуру, в которую в будущем можно будет подключать разные recommendation models и сравнивать их поведение на одинаковом пользовательском сценарии.

## Главный пользовательский сценарий

Пользователь заходит на сайт как гость, выбирает модель рекомендаций, генерирует плейлист и слушает треки.

Во время прослушивания система фиксирует пользовательские действия:

- like
- dislike
- skip
- listen

Эти данные сохраняются как interaction history и позже могут использоваться для обучения и оценки рекомендательных моделей.

## MVP ограничения

На этапе MVP:

- используется только одна recommendation model — random
- музыкальный каталог строится на subset датасета Last.fm
- backend возвращает только metadata треков
- воспроизведение происходит через YouTube IFrame API
- реальный ML pipeline пока отсутствует

## Архитектурная идея

Проект разделен на несколько независимых слоев:

### Backend API
Отвечает за:

- создание guest user
- генерацию playlist
- запись interaction events

### Recommendation Layer
Изолированный слой рекомендательной логики.

Сейчас содержит только random recommender, но позже должен позволять легко подключать новые алгоритмы.

### Frontend Playback Layer
Frontend отвечает за:

- поиск треков в YouTube
- воспроизведение
- управление очередью
- отправку пользовательских действий на backend

## Главный архитектурный принцип

Ядро системы — это не аудиоплеер.

Ядро системы — это контракт:

(user_id, model_name) → recommended tracks

и логирование поведения пользователя:

(user_id, track_id, model_used, interaction_type)

Именно вокруг этого будет строиться дальнейшее развитие recommendation platform.
