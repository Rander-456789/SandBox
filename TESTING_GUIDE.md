# RecSys Music Sandbox — Manual Testing Guide

> **Статус MVP**: Core реализован, стабилизирован, готов к ручному тестированию.

---

## Подготовка

### 1. Запуск Backend

```bash
cd backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**Проверка**: открой `http://localhost:8000/api/health` — должен вернуть `{"status": "ok"}`.

### 2. Запуск Frontend

```bash
cd frontend
npm run dev
```

**Проверка**: открой `http://localhost:3000` — должен показать лендинг с кнопкой «Continue as Guest».

## Тест-кейсы

### TC-01: Guest Auth Flow

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть `http://localhost:3000` | Лендинг с заголовком, кнопка «Continue as Guest» |
| 2 | Нажать «Continue as Guest» | Редирект на `/dashboard`, отображается «Session» |
| 3 | Проверить localStorage | `recsys-session` содержит `userId` и `activeModel: "random"` |
| 4 | Перезагрузить страницу | Сессия сохраняется, userId не меняется |

### TC-02: Playlist Generation

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Нажать «Generate Playlist» | Кнопка меняется на «Generating…» (disabled) |
| 2 | Дождаться генерации | Появляется плеер с названием трека и исполнителем |
| 3 | Проверить track counter | «Track 1 of 10» |
| 4 | Нажать «Generate Playlist» повторно (без двойного клика) | Старый плейлист заменяется новым, индекс сбрасывается на 1 |

**Edge Case**: нажать Generate дважды быстро → второй клик игнорируется (double-click guard).

### TC-03: YouTube Search

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Сгенерировать плейлист | Для каждого трека backend ищет YouTube videoId |
| 2 | Проверить консоль | Только `console.warn` если трек не найден; нет cache HIT/MISS спама |
| 3 | Нажать Next несколько раз, потом Previous | Кеш работает — при возврате к треку YouTube search не повторяется |

### TC-04: Hidden Playback

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть DevTools → Elements | `<div>` YouTube плеера скрыт: `width:0, height:0, opacity:0, z-index:-1` |
| 2 | Проверить iframe | Один `<iframe>` YouTube, без controls, без related videos |

### TC-05: Play/Pause

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Сгенерировать плейлист | Трек загружен (cued), но НЕ играет — `isPlaying = false` |
| 2 | Нажать ▶ (Play) | Начинается воспроизведение, иконка меняется на ⏸ |
| 3 | Нажать ⏸ (Pause) | Воспроизведение останавливается, иконка меняется на ▶ |
| 4 | Нажать ▶ снова | Воспроизведение возобновляется |

**Edge Case**: многократное нажатие Play/Pause → не должно быть ошибок, состояние корректно.

### TC-06: Autoplay (первый Play разблокирует)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Сгенерировать плейлист | Трек cued, не играет |
| 2 | Нажать Next | Трек 2 загружен (cued), не играет |
| 3 | Нажать ▶ (Play) на треке 2 | Начинается воспроизведение (`hasStarted = true`) |
| 4 | Дождаться окончания трека 2 | **Auto-next**: трек 3 начинает играть автоматически |
| 5 | Нажать Previous | Трек 2 начинает играть автоматически (autoplay разблокирован) |

### TC-07: Auto-Next (organic playback)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Запустить воспроизведение | Играет трек N |
| 2 | Дождаться окончания трека | Автоматически включается трек N+1 |
| 3 | Проверить консоль | **Нет** skip-события при auto-next |
| 4 | Проверить `/api/interactions/` | **Нет** записи `skip` для auto-next перехода |

### TC-08: Listen Events (dedup)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Запустить воспроизведение трека 1 | Отправляется ровно 1 `listen` interaction |
| 2 | Нажать Pause, затем Play | **Нет** повторного `listen` — второй PLAYING не отправляет listen |
| 3 | Перейти на трек 2 | Отправляется ровно 1 `listen` для трека 2 |
| 4 | Вернуться на трек 1 | **Нет** повторного `listen` для трека 1 (уже был в этой сессии) |
| 5 | Сгенерировать новый плейлист | Interaction memory сброшена — listen для тех же треков отправится заново |

### TC-09: Like / Dislike / Skip

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Нажать ♥ (Like) на треке 1 | Отправляется `like`, трек НЕ переключается |
| 2 | Нажать ♥ повторно | **Нет** второго `like` (dedup guard) |
| 3 | Нажать 💔 (Dislike) на треке 1 | Отправляется `dislike`, трек НЕ переключается |
| 4 | Нажать ⏭ (Next) | Отправляется `skip` + переход на трек 2 |
| 5 | Проверить: на треке 2 нажать Next | Отправляется `skip` для трека 2 |

### TC-10: End of Playlist

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Запустить воспроизведение, дойти до последнего трека | Играет «Track 10 of 10» |
| 2 | Дождаться окончания последнего трека | Плеер останавливается |
| 3 | Проверить UI | «Playlist finished» в плеере + в основном блоке |
| 4 | Проверить кнопки | Все кнопки неактивны (disabled) |
| 5 | Проверить консоль | Нет ошибок, нет бесконечного next-цикла |
| 6 | Нажать «Generate Playlist» | Плейлист сбрасывается, новый генерируется, кнопки активны |

**Edge Cases**:
- Если на последнем треке произошла ошибка YouTube → тоже `markPlaylistFinished()` (без auto-skip за пределы)
- Если на последнем треке нажать Next → кнопка disabled, ничего не происходит

### TC-11: Rapid Track Switching

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Запустить воспроизведение | Играет трек 1 |
| 2 | Быстро нажать Next 5 раз подряд | Переход к треку 6, без ошибок |
| 3 | Проверить: предыдущие videoId не применяются | `activeTrackIdRef` guard предотвращает гонки |
| 4 | Быстро нажать Previous 3 раза | Переход к треку 3, без дублирования |

### TC-12: Missing Video / Failed Search

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Дождаться трека без videoId (если такой есть в выдаче) | `console.warn` «Video not found» |
| 2 | Проверить: следующий трек | Auto-skip на следующий (если autoplay разблокирован) |
| 3 | Либо: ручной Next | Работает корректно |

### TC-13: Multiple Play/Pause (Stress)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Запустить воспроизведение | Играет |
| 2 | Нажать Pause/Play 10 раз быстро | Состояние корректное, нет ошибок |
| 3 | После паузы: autoplay всё ещё работает | При следующем ENDED → auto-next |

### TC-14: Loading UX

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Сгенерировать плейлист | При первом треке: Play кнопка показывает спиннер (buffering) |
| 2 | Проверить navigation | Prev/Next/Like/Dislike disabled во время loading |
| 3 | После загрузки | Спиннер исчезает, кнопки активны |
| 4 | Нажать Next | Кратковременный спиннер + кнопки disabled |

## Final Verification Checklist

- [ ] **guest auth** — создание гостя, хранение в localStorage, survivability при reload
- [ ] **playlist generation** — генерация через `POST /api/playlist/generate`, отображение треков
- [ ] **YouTube search** — поиск через `GET /api/youtube/search`, runtime-кеш в памяти
- [ ] **hidden playback** — плеер скрыт, без UI-элементов YouTube
- [ ] **play/pause** — корректное переключение, иконка меняется
- [ ] **autoplay** — после первого Play нажатия автовоспроизведение разблокировано
- [ ] **auto-next** — organic переход по ENDED, без skip-события
- [ ] **listen events** — ровно 1 на трек за сессию, dedup через Set
- [ ] **like/dislike/skip** — корректная отправка, dedup для like/dislike
- [ ] **duplicate protection** — listen/like/dislike не дублируются в рамках сессии
- [ ] **playlist ending** — остановка на последнем треке, «Playlist finished», кнопки disabled
- [ ] **no infinite loops** — ENDED на последнем треке → stop (не auto-next)
- [ ] **race conditions** — rapid switching не вызывает гонок (activeTrackIdRef guard)
- [ ] **no debug spam** — консоль чистая, только warn/error при реальных проблемах
- [ ] **clean UI** — нет debug-панелей, нет сырых videoId, нет cache counters
- [ ] **loading feedback** — спиннер при buffering, disabled controls во время загрузки
